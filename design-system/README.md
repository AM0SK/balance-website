# design-system

Пакет дизайн-системи Balance. Зберігати разом із проєктом — цього достатньо, щоб у будь-який момент (навіть у новій сесії, без історії листування) коректно зверстати сайт або зібрати бібліотеку у Figma.

| Файл | Що це |
|---|---|
| `DESIGN-SYSTEM.md` | головний документ: структура застосунку, всі компоненти, стани, правила |
| `tokens.css` | CSS-змінні обох тем — підключати першим у проєкті сайту |
| `tokens.json` | ті самі токени у форматі Tokens Studio → імпорт у Figma |
| `balance-mockup.html` | візуальний еталон: 6 екранів, перемикач тем. Відкривається подвійним кліком |

## Швидкий старт для вёрстки

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Nunito:wght@200..1000&display=swap" rel="stylesheet">
<link rel="stylesheet" href="design-system/tokens.css">
```

Далі всі кольори, розміри й ефекти брати **тільки** зі змінних `--*`, не хардкодити значення.

## Швидкий старт для Figma

Плагін Tokens Studio → Import → `tokens.json` → створити Themes `Light` і `Dark` → Apply.

## Затверджені параметри

Типографіка та ефекти зафіксовані клієнтом 18.07.2026 і змінам без запиту не підлягають:
розмиття карток 5px · таб-бар 10px · фон модалки 5px · висота кнопок 40px ·
товщина кільця 6px · точка градієнта фону 65% · непрозорість карток 5% (темна) / 100% (світла) ·
підсвітка продукту 20% (світла) / 15% (темна).
