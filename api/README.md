# Balance API

PHP 8.4 + MySQL 8.0 без Composer — на shared-хостингу Hostia немає ні SSH,
ні можливості ставити залежності. Використовуються лише PDO і `hash_hmac`.

## Розгортання

1. **База даних.** У панелі Hostia створити MySQL-базу і користувача.
   Через phpMyAdmin виконати спершу `schema.sql`, потім `seed.sql`.

2. **Бот.** У Telegram створити бота через [@BotFather](https://t.me/BotFather),
   отримати токен. Ним перевіряється підпис `initData` — без нього авторизації немає.

3. **Конфіг.** Скопіювати `config.example.php` у `config.php` і заповнити.
   `config.php` у `.gitignore` — у репозиторій не потрапляє, заливається вручну.

   > `dev_mode` на бойовому сервері **має бути `false`**. З `true` застосунок
   > пускає будь-кого без Telegram під тестовим користувачем.

4. **Перевірка.** Відкрити `https://balance.amosk.com.ua/api/check.php` —
   покаже, що саме не так. Після успішної перевірки файл видалити з сервера.

## Маршрути

Авторизація — заголовок `X-Telegram-Init-Data` зі значенням
`window.Telegram.WebApp.initData`. Без валідного підпису — `401`.

| Метод | Шлях | Призначення |
|---|---|---|
| `GET` | `/api/bootstrap` | усе для старту: довідники, профіль, дані дня |
| `PATCH` | `/api/profile` | ім'я, фото, денний ліміт калорій, цілі |
| `PUT` | `/api/consumption` | скільки продукту спожито; `units: 0` прибирає |
| `POST` | `/api/workouts` | додати тренування |
| `DELETE` | `/api/workouts?id=` | видалити тренування |
| `PUT` | `/api/steps` | кроки за день (перезапис) |
| `PUT` | `/api/measurements` | замір за день (перезапис) |

`GET /api/bootstrap` навмисно віддає все одним запитом: у мобільній мережі
кожен зайвий roundtrip помітний.

## Модель даних

Головне, що варто розуміти: **бюджет категорії спільний для всіх її продуктів**.
У БД зберігається щільність продукту (ккал і білок на 1 г або 1 шт), а не
грамівка порції — грамівка є похідною і змінюється протягом дня:

```
ліміт(продукт) = спожито(продукт) + залишок_бюджету_категорії / ккал_на_одиницю
```

Тому з'їдені 52 г овочів знижують показаний ліміт грибів із 600 до 548 г.
Математика живе на клієнті (`src/lib/ration.ts`, покрита тестами), сервер
зберігає сирі дані й валідує записи.

## Тести

```
php api/tests/auth_test.php
```

Перевіряють підпис Telegram: коректний приймається, підроблений і
протермінований відхиляються, підміна `user_id` ламає хеш.

## Що не перевірено

Код проти **справжньої MySQL не запускався** — локально немає ні MySQL, ні
Docker. Перевірено на SQLite: маршрутизація, авторизація, валідація, форми
відповідей. Не перевірено: діалект-специфічний `ON DUPLICATE KEY UPDATE`
і зовнішні ключі. Перший запуск на сервері варто починати з `check.php`.
