## Що це за проєкт

Веб-застосунок **Balance**, що відкривається за посиланням як Telegram mini-app.
**Тільки мобільна версія** (ширина ~360–430 px). Мова інтерфейсу — українська.

Трекер: раціон за категоріями обміну продуктів (А–Ж), кроки, тренування, вага, заміри тіла.

## Стек

React 19 + TS + Vite (збірка в статику) · PHP 8.4 без фреймворку · MySQL 8.0 ·
деплой GitHub Actions → FTP.

Hostia — shared-хостинг: немає Node, SSH і Composer. Laravel свідомо не взято —
~185 МБ вільного місця, vendor/ з'їв би значну частину без користі.

## Головне про модель даних

**Бюджет категорії спільний для всіх її продуктів.** Зберігається щільність
продукту (ккал і білок на 1 г або 1 шт), а не грамівка порції — грамівка
похідна й змінюється протягом дня:

```
ліміт(продукт) = спожито(продукт) + залишок_бюджету_категорії / ккал_на_одиницю
```

Тому з'їдені 52 г овочів знижують ліміт грибів із 600 до 548 г. Підтверджено
скріншотами реального застосунку, закріплено тестом.

**Білок не має цілі** — це лічильник. Натомість діє гарантований нижній поріг:
за будь-якого вибору продуктів користувач набере не менше 50 г. На лімітах
нижче ~1425 ккал бюджет автоматично перерозподіляється в бік білкових категорій.

Математика — `src/lib/ration.ts`, 18 тестів.

## Що лишилось, щоб сайт запрацював

1. Створити базу в панелі Hostia, виконати `api/schema.sql`, потім `api/seed.sql`.
2. Створити бота через @BotFather, узяти токен.
3. Заповнити `api/config.php` (шаблон — `config.example.php`), залити вручну.
   `dev_mode` **має бути `false`**.
4. Додати секрети в GitHub → Settings → Secrets → Actions:
   `FTP_SERVER`, `FTP_USERNAME`, `FTP_PASSWORD`, `FTP_SERVER_DIR`.
5. Відкрити `/api/check.php`, потім видалити файл із сервера.

## Відомі обмеження

- **Бекенд проти справжньої MySQL не запускався** — локально немає MySQL і
  Docker. Перевірено на SQLite: маршрути, авторизація, валідація, форми
  відповідей. Не перевірено `ON DUPLICATE KEY UPDATE` і зовнішні ключі.
- **Дані продуктів — реконструкція** зі скріншотів; ліміти розходяться з
  реальними на кілька відсотків. Правити в `api/seed.sql`.
- Сума бюджетів категорій ≈1733 проти заявлених 1766 ккал — лишено як є.
- **Apple Health недоступний**: mini-app працює у WebView без доступу до
  HealthKit. Кроки — ручний ввід.
- Токен GitHub акаунта AM0SK потребує права `workflow`, інакше пуш
  `.github/workflows/*` відхиляється.

## Ключові рішення (не переглядати без запиту клієнта)

- **Шрифт** — Nunito (variable 200–1000), підключати з Google Fonts.
- **Без зеленого в базовій палітрі.** Єдиний акцент — градієнт `#343147 → #020202`.
  Зелений використовується **лише** як семантика стану «продукт почато».
- **Світла тема:** сірий папір `#F4F4F6` + білі картки з м'якою тінню.
  **Темна тема:** фон — вертикальний градієнт `#343147 → #020202` (точка виходу в чорний 65%), картки напівпрозорі (5%) з `backdrop-blur 5px`.
- **Таб-бар — 4 вкладки:** Головна · Раціон · Вправи · Кроки. Активна вкладка позначається кольором, **без заливки-пігулки**. Заміри тіла перенесені з таб-бару в Налаштування.
- **Шапка (лого + аватар + шестерня) закріплена** на кожному екрані; заголовок H1 і кнопка «+ Додати» — усередині зони прокрутки.
- **Не все в картках:** шкали БЖУ, розділ «Акаунт» та історія тренувань/кроків — просто на фоні з розділювальними лініями.
- **Раціон:** одна картка = одна категорія, продукти всередині розділені лініями; чип-літера лише в шапці картки.
- **Стани рядка продукту:** зелена підсвітка — почато; бурштинова `#CC9A3A` — денний ліміт вичерпано. Заливка градієнтом зліва прозоре → праворуч колір.

## Дві пастки реалізації (перевірено на практиці)

1. Контейнер зі скролом мусить мати внутрішні відступи
   (`padding: 8px 8px 92px; margin: -8px -8px 0`), інакше **тінь карток обрізається** з боків і згори.
2. Маска кільця-діаграми потребує **`closest-side`**:
   `radial-gradient(circle closest-side …)`. Без цього радіус рахується до кута квадрата
   і товщина обводки виходить майже втричі більшою за задану.
